# Entregaveis de Competicao

## Entregaveis atuais

- README com contexto, aviso clinico e execucao local.
- API FastAPI com endpoints minimos.
- Motor de regras explicavel.
- Seed sintetico.
- Testes basicos.
- Documentacao de produto, arquitetura, dados, regras, LGPD, validacao, demonstracao e pitch.

## Demonstracao funcional

A demonstracao deve mostrar um fluxo simples:

1. Carregar dados sinteticos.
2. Exibir alertas priorizados.
3. Abrir explicacao de um alerta.
4. Registrar validacao humana.
5. Registrar acao realizada.
6. Mostrar dashboard territorial.

## Narrativa problema-solucao

Problema: equipes de APS precisam priorizar busca ativa, contatos e adesao em cenario de informacao fragmentada.

Solucao: uma fila operacional explicavel que organiza questionarios, calcula risco por regras e apoia decisao humana.

## Impacto social esperado

- Menor atraso operacional na busca ativa.
- Melhor visibilidade de contatos e barreiras.
- Planejamento territorial mais transparente.
- Apoio a equipes com alta carga de trabalho.
- Base para piloto responsavel com governanca de dados.

## Limites clinicos e regulatorios

O MVP nao realiza diagnostico, triagem clinica definitiva, prescricao, classificacao epidemiologica oficial ou integracao com bases reais. Alertas sao recomendacoes operacionais que exigem validacao humana e protocolos locais.

## Evolucao pos-competicao

- Prototipo visual para equipe de APS.
- Validacao com profissionais de saude.
- Ajuste de regras com especialistas.
- Modelagem de seguranca, auditoria e consentimento.
- Integracao futura e controlada com sistemas oficiais, somente apos autorizacao institucional e avaliacao juridica.
- Estudo piloto com indicadores de processo e seguranca.
